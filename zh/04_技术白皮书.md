## 摘要

本白皮书回答一个问题：**Valhalla 作为 AI 孵化运行时，与 Transformer 是什么关系？实验上证明了什么、没证明什么？**

**结论摘要**:

1. **Tier A**（Valhalla 结构 → patch Qwen2.5-0.5B）：在 48 题 strict 基准上，**无净准确率提升**；Valhalla triad patch（-2.08 pp）**显著优于** 同强度 random patch（-14.58 pp），输出变化率 79–92%。
2. **Tier B**（`valhalla_native_qa`，无 HF forward）：48 题准确率 22.92%→25.00%（triad + 语料，+2.08 pp）；GSM/MATH 数值子集 8/9；**整体远低于** Transformer baseline ~69%。
3. **KD 对照**（Qwen2.5-3B → 0.5B lm_head SFT）：**+22.92 pp**（68.75%→91.67%）— 代表 **有监督梯度范式** 在同基准上的上限，与 Valhalla **无 backprop 范式** 并列，不可混称为同一类「孵化增益」。

**非目标**: 宣称 Valhalla 已在通用 benchmark 上替代 Transformer。

## 1. 术语与分层

| 术语 | 定义 |
|------|------|
| **Incubation（孵化）** | 语料/问题经 Valhalla ingress → 结构演化 → 可测行为变化 |
| **Tier A（代理）** | 结构 export 为 `model_patch`，作用于外部 Transformer 权重 |
| **Tier B（原生）** | 结构演化 + 原生解码；不调用 `model.generate()` |
| **strict MCQ** | 必须提取 `ANSWER: X` 或末行单字母；禁止子串 fallback |
| **Body** | `hub` / `tile` / `stemcell` / `triad` — export 时启用的子系统组合 |

## 3. Tier A 实现与结果

### 3.1 Patch 几何

`apply_valhalla_patch`（Python）:

- 4 组 `input_layernorm.weight` 按 `quad_layer_scales` 缩放
- prompt token embedding 加 `patch_vector` 投影扰动
- strength=0.08（默认）

**协议**: 每题 **fresh load** 模型；禁止跨题累积 patch（否则 collapse）。

### 3.2 结果表（strict_incubation_standard_20260616_0802）

| Arm | Acc after | Gain (pp) | Change rate |
|-----|-----------|-----------|-------------|
| triad_c1 | 66.67% | **-2.08** | ~88% |
| stemcell_c1 | 64.58% | -4.17 | ~92% |
| hub_c1 | 58.33% | -10.42 | ~88% |
| prompt_only | 62.50% | -6.25 | — |
| random_patch | 54.17% | **-14.58** | — |

### 3.3 方法学更正

早期报告 Hub/Stem **+4.17 pp** 源于 `score_answer` 宽松模式：

```python
# 已禁止（strict 默认）:
hit = ref_letter.lower() in low  # "B" in "about" → 假阳性
```

**权威更正**: `reports/VALHALLA_EXPERIMENT_RECORD_CORRECTIONS_20260616.md`

### 3.4 可检验声明（Tier A）

| 声明 | 证据 | 状态 |
|------|------|------|
| Patch 改变输出 | change_rate 79–92% | ✓ |
| 结构优于随机 | triad -2.08 vs random -14.58 | ✓ |
| Strict 净 acc 提升 | 所有 Valhalla 臂 ≤ 0 | ✓ 无提升 |
| 外部题无 Valhalla 幻觉 | 0 关键词命中 | ✓ |

## 4. Tier B 实现与结果

### 4.1 原生解码栈

**二进制**: `valhalla_native_qa`  
**源码**: `manifestsys/hub-f64/src/native_qa.rs`

```
corpus[] + question → TriadSession.feed (NormalizedF64)
                   → finalize_with_cycles(N)
                   → hub_prefs + patch_vector + tile/stem signatures
                   → decode_mcq | decode_numeric | decode_open
```

**v2 数值解码** (`decode_numeric`):

- `try_direct_math`: multiply / divide / sqrt / linear equation
- `try_word_problem`: half-as-many, eggs/market, total cost, give/buy
- `cortex_numeric_candidates`: Difference, Ratio, sum
- Fate 加权 `score_memories`

### 4.2 结果表（tier_b_native_standard_20260616_0847）

| Arm | Correct | Acc | Δ vs baseline |
|-----|---------|-----|---------------|
| baseline_no_corpus | 11/48 | 22.92% | — |
| triad_c1 | 12/48 | 25.00% | +2.08 pp |
| triad_c10 | 12/48 | 25.00% | +2.08 pp |

**数值子集（triad_c1）**:

| ID | Ref | Decode path | Correct |
|----|-----|-------------|---------|
| GSM_01 | 72 | word_problem | ✓ |
| GSM_02 | 18 | word_problem | ✓ |
| GSM_03 | 18 | word_problem | ✓ |
| GSM_04 | 28 | word_problem | ✓ |
| MATH_01 | 391 | direct_math | ✓ |
| MATH_02 | 12 | direct_math | ✓ |

### 4.3 结构收敛观察

`triad_c10` ≡ `triad_c1` 在 acc 与多数结构指标上 — **与 Tier A 一致**：Tile/Stem 1-cycle Fate 收敛，多轮无额外增益。

## 5. KD 对照实验

### 5.1 协议

- **Teacher**: Qwen/Qwen2.5-3B-Instruct，48 题生成 label
- **Student**: Qwen/Qwen2.5-0.5B-Instruct，**仅 lm_head** 可训练
- **训练**: fp32，3 epoch，lr=1e-5，grad clip 1.0
- **脚本**: `tools/valhalla_model_bridge/run_kd_distill.py`

### 5.2 结果

| Metric | Value |
|--------|-------|
| Acc before | 68.75% (33/48) |
| Acc after | **91.67%** (44/48) |
| Gain | **+22.92 pp** |
| Improved / Regressed | 11 / 0 |

### 5.3 与 Valhalla 并列解读

| 维度 | Valhalla Tier A | KD |
|------|-----------------|-----|
| 梯度 | 无 | 有 |
| Teacher | 无 | 3B |
| 48Q strict Δ | -2.08 pp (best) | +22.92 pp |
| 部署叙事 | 结构 checkpoint | 微调权重 |

## 6. 系统边界与未来工作

### 6.1 当前边界

| 边界 | 说明 |
|------|------|
| 样本量 | 48 题，±1 题 ≈ ±2.08 pp |
| Tier B 生成 | 检索+规则，非 autoregressive |
| 语料覆盖 | medium 56 行，非 GSM 全量 |
| 多轮 cycle | 无增益，需新 ingress 或 loss 信号 |

### 6.2 路线图（技术）

1. Tier B `speak.rs` 深度集成 — Fate 驱动生成式片段
2. 扩展 benchmark 200+ 题 + bootstrap CI
3. Tier A embedding-only patch ablation
4. 垂直 POC 固定域语料 + native 解码专精
